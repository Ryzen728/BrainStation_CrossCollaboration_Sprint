This project addressed the need for more human oversight over generative artificial intelligence (GenAI) technology.

Objective: <br>
Work with a cross-disciplinary team (UX designer, software designers, data scientist) to analyze a business problem and develop a thoughtful, strategic solution. 

Requirements:
1. Build a functional MVP that incorporates technical contributions from all team members.
2. Create and deliver a stakeholder-ready pitch, supported by data-driven insights and clear recommendations.
3. Employ advanced, reproducible analysis yielding validated and actionable insights that directly support decisions (Data Analysis & Insight Generation)

The Challenge: <br>
How might we help distributed teams turn AI-generated outputs into trustworthy, actionable decisions without losing human oversight?

The Solution: <br>
An app that users GenAI to generate an email from a prompt of a few key words. The user would subsequently select from three options to evaluate the acceptability of the generated email.
Responses from the user could be used to refine the GenAI model. The iniital pre-training from the model would come from "ethical" data sources. Further transfer learning training of the model would come from a dataset of customer information from around the world. 
The example use case was an uber driver.

Criteria Satisfaction:
1. The analysis was not statistically advanced, but it demonstrated a human oversight and thoughtfulness that is not considered often enough. 
2. The insights are actionable and can directly support decision-making. It is a more costly approach, however.

Contributions:
1. Software Engineers: The app solution concept. 
2. UX Designer: UX design, including the figma site.
3. Data Scientist (myself): <br>
  Conceptualization: Recognizing the importance of the ethical quality of the training data and not accepting the convential ready-made pre-training data, which is not transparent nor known to emphasize ethical values. <br>
  Data Sourcing: Kaggle.com <br>
  Data Cleaning: Checking for null and missing values <br>
  Feature Engineering: Consolidating records into count values and merging datasets for EDA and data visualization. <br>
  Exploratory Data Analysis (EDA): Verifying how the ethical datasets address ethical principles, and noting how imbalances in the data will add its own bias. <br>
  Data Visualization: Representing the count data as bar charts, merging count data with a geospatial dataset to present the geographical distribution of data points. <br>
  Presentation: Focusing on the need to ensure the source data is as ethical as possible, rather than showcasing statistical analysis skills with questionable underlying data. <br>
